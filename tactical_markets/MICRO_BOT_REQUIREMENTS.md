## Overview

The tactical_markets_trading bot (trading execution layer) consumes tactical sector rotation and momentum theses from MICRO (signal generation) to:
1. **Generate pre-market trade opportunities** (3–5 actionable theses by 6:30 AM ET)
2. **Define position hold windows and exit rules** (target/stop/timeout)
3. **Provide strategy context** for live execution and post-trade analysis

This document specifies what MICRO must provide for the bot to execute Phase 1A MVP and beyond.

## Phase 1A MVP Requirements (Blocking)

### API Endpoint: `GET /api/theses`

**Availability:** Daily, 6:30 AM ET (premarket, before market open at 9:30 AM ET)

**Request Parameters (Optional):**
- `limit=5` — Return top N theses (default: 5, max: 10)
- `min_confidence=60` — Filter to theses with confidence ≥ threshold (default: 60)

**Response Schema:**

```json
{
  "timestamp": "2026-05-13T06:30:00Z",
  "theses": [
    {
      "thesis_id": "uuid",
      "symbol": "XLE",
      "direction": "BUY",
      "entry_price": 91.30,
      "stop_price": 89.50,
      "target_price": 94.00,
      "hold_window_hours": 48,
      "confidence": 75,
      "reasoning": "Sector rotation: relative strength XLE vs XLK 3.9% (top 10%). Dollar weakness supports energy. Hold 48h within quarter.",
      "strategy_family": "sector_rotation",
      "backtest_sharpe": 0.92,
      "backtest_win_rate": 0.58
    },
    {
      "thesis_id": "uuid",
      "symbol": "IWM",
      "direction": "SELL",
      "entry_price": 192.50,
      "stop_price": 195.00,
      "target_price": 189.00,
      "hold_window_hours": 24,
      "confidence": 68,
      "reasoning": "Gap fill: overnight repricing favors large-caps over small-caps; IWM overbought relative to SPY. Hold 24h.",
      "strategy_family": "gap_fill",
      "backtest_sharpe": 0.68,
      "backtest_win_rate": 0.52
    }
    // ... (up to 5 theses, ordered by confidence descending)
  ]
}
```

**Field Definitions:**

- **thesis_id:** Unique UUID for this thesis. Used in bot trade logging to trace back to originating signal.
- **symbol:** Single ticker (e.g., "XLE", "IWM", "SPY"). Must be liquid (min 1M shares daily volume).
- **direction:** `"BUY"` or `"SELL"`. Bot interprets as market entry direction.
- **entry_price:** Suggested entry price (limit order price). Bot may adjust ±5 bps based on current market.
- **stop_price:** Stop-loss level. Bot exits position if price touches/crosses this level.
- **target_price:** Profit-taking level. Bot exits position if price reaches this level.
- **hold_window_hours:** How long thesis is expected to work (hours). Bot auto-exits at market if position still open when window expires. Typical: 24–120 hours.
- **confidence:** 0–100 score. Higher = more conviction. Bot uses to scale position size (high confidence → larger position, within limits).
- **reasoning:** Plain-English explanation of thesis. Logged in bot trade history for post-trade analysis and user review.
- **strategy_family:** Categorical classification (e.g., "sector_rotation", "gap_fill", "volatility_mean_reversion", "carry", "breakout", "momentum"). Used by bot for strategy allocation + performance tracking.
- **backtest_sharpe:** Historical Sharpe ratio for this strategy family (e.g., 0.92 for sector_rotation over 2015–2026). Informational; helps user evaluate thesis edge.
- **backtest_win_rate:** Historical win rate for this strategy family (e.g., 0.58 = 58% win rate). Informational; used in bot daily reporting.

**Thesis Count & Prioritization:**

- Return up to 5 theses per day (default `limit=5`).
- Order by confidence descending (highest confidence first).
- Filter to theses with confidence ≥ min_confidence (default: 60).
- If fewer than 5 theses pass filter, return what's available (may return 0–5).

**Latency SLA:** Response completes within 2 seconds (95th percentile). If timeout >5s, bot logs warning and uses cached theses.

**Freshness SLA:** Data freshness <1 hour old at consumption time. If timestamp >1 hour old, bot logs alert and disables new trade execution (existing positions continue per their rules).

## Data Quality Requirements

### Validation Rules (MICRO Should Enforce)

1. **Symbol Liquidity:** Theses must be for symbols with min 1M shares daily volume. Exclude illiquid/penny stocks.
2. **Entry/Stop/Target Spread Reasonableness:**
   - For BUY: entry < target AND entry > stop. Spread (target - entry) / entry should be 1–5% (typical thesis horizon 24–120h).
   - For SELL: entry > target AND entry < stop. Same spread logic.
   - Reject theses with <0.5% spread (too tight; likely to hit stop before target) or >10% spread (unrealistic for short hold window).
3. **Hold Window Bounds:** 24 hours ≤ hold_window ≤ 240 hours (10 days max for Phase 1A).
4. **Duplicate Symbols:** If multiple theses for same symbol in same run, keep highest-confidence thesis; suppress others (log as "filtered: duplicate symbol").
5. **Confidence Score:** Must be 0–100 integer. Missing = reject thesis.

### Conflict Resolution

If theses conflict (e.g., two theses for XLE, one BUY @ 91 and one SELL @ 92), keep highest-confidence thesis. Log lower-confidence theses as "filtered: conflicting signal".

## Phase 1B Enhancement (Optional but Recommended)

### Intraday Thesis Refresh Support

**Trigger:** If market conditions shift significantly intraday (e.g., new news, regime change), MICRO optionally refreshes thesis list mid-day:

- Endpoint: `GET /api/theses?intraday=true`
- Response includes: new theses + status of morning theses (unchanged, updated confidence, expired)
- Bot processes refresh: adds new theses to dashboard, updates hold windows if confidence changed, marks expired theses as stale

**Use Case:** Allows bot to adapt Tier 2 single-stock picks and intraday momentum trades to market evolving conditions.

**Implementation:** Optional for Phase 1B. If not implemented, bot uses pre-market theses through end-of-day.

## Failure Modes & Graceful Degradation

### Endpoint Unavailable (Timeout or 5xx Error)

1. **If unavailable <1 hour:** Bot uses last-known theses (cached for 24 hours). Logs warning: "MICRO theses stale; using cache from [time]."
2. **If unavailable 1–2 hours:** Bot continues managing existing positions per rules (target/stop/hold-window). No new trade opportunities generated. Dashboard alerts user: "MICRO signal unavailable; no new theses generated."
3. **If unavailable >2 hours:** Bot pauses all operations; manual intervention required.

### Stale Data (timestamp >1 hour old)

Same as unavailable scenario above. Freshness is critical; stale theses = unreliable signal.

### Invalid Response (Schema Mismatch or Missing Fields)

Bot logs error, uses cached theses, alerts user on dashboard. Requires manual investigation + fix.

### Empty Thesis List (0 theses returned)

Valid scenario. Bot manages outstanding positions per rules; no new opportunities. Not an error; logged as "no theses generated today".

## Testing & Validation

**Pre-Integration Testing (MICRO team):**
1. Endpoint returns 3–5 valid theses daily by 6:30 AM ET
2. Theses pass validation (liquidity, entry/stop/target spreads, hold window bounds)
3. Confidence scores are realistic (not all 99, not all 50)
4. Latency <2s under normal load
5. Timestamps accurate (within seconds of current time)

**Integration Testing (Bot + MICRO):**
1. Bot polls endpoint at 6:30 AM ET; receives 3–5 valid theses
2. Dashboard displays theses with entry/stop/target + reasoning
3. User clicks "Execute"; bot submits limit orders within 5 bps of market
4. Position holds for duration specified in `hold_window_hours`; auto-exits on target/stop/timeout
5. Trade log includes thesis_id, strategy_family, backtest metrics; backtesting correlates strategy performance
6. 20+ theses executed in Phase 1A; win rate ≥51%, Sharpe ≥0.5
